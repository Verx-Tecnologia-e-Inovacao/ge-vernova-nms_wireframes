# Glossario de Termos Tecnicos

Este glossario reune as siglas e termos tecnicos utilizados nos documentos de wireframe do NMS (Network Monitoring System). O objetivo e facilitar a compreensao do conteudo por profissionais que nao sao familiarizados com o dominio de automacao de subestacoes eletricas e protocolos de rede industrial.

Para convencoes visuais dos wireframes (simbolos ASCII, componentes de interface, padroes de layout), consulte o documento `00-navegacao-global.md`.

---

| Termo | Definicao |
|---|---|
| **Access Point (AP)** | Ponto de conexao de rede de um IED. Cada AP conecta o dispositivo a uma sub-rede especifica (ex: S1 conecta ao Station Bus, P1 conecta ao Process Bus). Um IED pode ter multiplos APs para se conectar a diferentes redes simultaneamente. |
| **ACK** | Abreviacao de Acknowledgement (reconhecimento). No contexto do NMS, e a acao de um operador confirmar que tomou ciencia de um alarme. Alarmes com `ack=false` sao nao reconhecidos e exigem atencao. |
| **API** | Application Programming Interface (Interface de Programacao de Aplicacoes). Conjunto de endpoints HTTP que o frontend do NMS utiliza para se comunicar com o backend, seguindo o padrao REST. A especificacao completa esta em `docs/api/swagger-nms-v1.0.1.yaml`. |
| **Bearer Token** | Mecanismo de autenticacao HTTP onde um token (JWT) e enviado no cabecalho `Authorization` de cada requisicao. O token e obtido no login e renovado automaticamente via refresh token. |
| **CID** | Configured IED Description (Descricao de IED Configurado). Arquivo XML no padrao IEC 61850 que descreve a configuracao de um unico IED. E uma variante do SCD focada em um dispositivo individual. O NMS aceita upload de arquivos CID alem de SCD. |
| **Control Block** | Bloco de controle definido no SCD que configura a publicacao de mensagens GOOSE por um IED. Cada control block possui um identificador (ex: GCB01, GoCB01), um dataset associado e parametros de rede (MAC, VLAN, sub-rede). A subestacao de exemplo possui 11 control blocks GOOSE. |
| **Data Attribute (DA)** | Atributo de dados no modelo IEC 61850. E o nivel mais granular da hierarquia de dados de um IED, representando um valor individual (ex: posicao de um disjuntor, valor de corrente, status de protecao). |
| **Data Point** | Ponto de dado monitorado pelo NMS. Cada IED possui um conjunto de data points que sao acompanhados em tempo real pelo motor de monitoramento. A contagem de data points monitorados e exibida no detalhe de cada IED. |
| **Dataset** | Conjunto de dados associado a um control block GOOSE ou stream SV. Define quais informacoes sao publicadas em cada mensagem. Expresso em numero de entradas (ex: 64 entradas significa 64 valores transmitidos por mensagem). |
| **Firmware** | Software embarcado em um IED. Indica a versao do sistema operacional e funcionalidades do dispositivo (ex: L90 v8.60, T60 v8.70). Exibido nos detalhes de cada IED para identificacao. |
| **GOOSE** | Generic Object Oriented Substation Event (Evento Generico Orientado a Objetos de Subestacao). Protocolo de comunicacao IEC 61850 para troca rapida de eventos entre IEDs, como comandos de protecao e sinalizacao de estado. Opera em Layer 2 (sem IP) com latencia muito baixa. A subestacao de exemplo possui 11 control blocks GOOSE. |
| **Grandmaster Clock** | Relogio de referencia principal em uma rede PTP. Todos os demais dispositivos sincronizam seu tempo com base neste relogio. Identificado por um `grandmasterId` unico. No NMS, o status do Grandmaster Clock e exibido na tela de Sincronismo Temporal. |
| **HSR** | High-availability Seamless Redundancy (Redundancia Transparente de Alta Disponibilidade). Protocolo de redundancia de rede definido na norma IEC 62439-3 que opera em topologia de anel, garantindo comunicacao sem perda de pacotes durante falhas de enlace. No NMS, esta pendente de definicao na API. |
| **IEC 61850** | Norma internacional que define protocolos de comunicacao para automacao de subestacoes eletricas. Estabelece o modelo de dados dos IEDs (Logical Devices, Logical Nodes, Data Attributes), os protocolos de troca de dados (GOOSE, SV, MMS) e o formato de configuracao (SCD/CID). E a base de todo o dominio do NMS. |
| **IEC 62439-3** | Norma internacional que define protocolos de redundancia de rede para aplicacoes industriais, incluindo HSR (redundancia em anel) e PRP (redes paralelas). Garante comunicacao sem perda de pacotes em subestacoes. |
| **IED** | Intelligent Electronic Device (Dispositivo Eletronico Inteligente). Equipamento digital instalado na subestacao que realiza funcoes de protecao, medicao ou controle. Exemplos na subestacao monitorada: L90_DIG (rele de protecao de linha), T60_DIG (rele de protecao de transformador), MU_360_1 e MU320E_LAB (merging units). |
| **IP** | Internet Protocol (Protocolo de Internet). Protocolo de rede Layer 3 usado para enderecamento e roteamento de pacotes. Algumas sub-redes da subestacao operam sobre IP (ex: rede administrativa W03), enquanto outras operam em Layer 2 (ex: Process Bus para GOOSE/SV). |
| **ISO 8601** | Padrao internacional para representacao de datas e horas. Formato utilizado pela API do NMS para todos os timestamps (ex: `2026-02-19T14:30:00Z`). O frontend converte para o formato local do usuario antes de exibir. |
| **Jitter** | Variacao na latencia de entrega de pacotes de rede. No contexto de Sampled Values, o jitter e medido em microsegundos (us) e indica a estabilidade da transmissao. Valores altos de jitter podem comprometer a qualidade das medicoes e gerar alarmes. |
| **JWT** | JSON Web Token. Padrao aberto para transmissao segura de informacoes entre partes como um objeto JSON assinado. No NMS, o JWT e emitido no login e contem as credenciais do usuario (incluindo o role: ADMIN, OPERATOR ou VIEWER). |
| **Layer 2** | Camada de enlace de dados do modelo OSI. Protocolos que operam em Layer 2 (como GOOSE e SV) comunicam-se diretamente por enderecos MAC, sem necessidade de enderecamento IP. Isso garante menor latencia, essencial para protecao em subestacoes. |
| **Logical Device (LD)** | Dispositivo logico no modelo IEC 61850. E um agrupamento funcional dentro de um IED que contem Logical Nodes. Um IED pode ter multiplos LDs (ex: L90_DIG possui 3 LDs). |
| **Logical Node (LN)** | No logico no modelo IEC 61850. Representa uma funcao especifica dentro de um Logical Device (ex: LTMS para gerenciamento de tempo, XCBR para disjuntor, LPDI para indicacoes digitais). Cada LN contem Data Attributes com valores monitoraveis. |
| **LTMS** | Logical Node for Time Management Supervision (No Logico de Supervisao de Gerenciamento de Tempo). No logico IEC 61850 presente em IEDs que expoe dados de sincronizacao temporal, incluindo TmAcc (acuracia em nanossegundos) e TmSrc (fonte de tempo). |
| **MAC** | Media Access Control (Controle de Acesso ao Meio). Endereco fisico unico de uma interface de rede, utilizado na comunicacao Layer 2. No contexto do NMS, os enderecos MAC identificam os destinos de mensagens GOOSE e streams SV (ex: `01:0c:cd:01:00:00`). |
| **Merging Unit (MU)** | Unidade de fusao. Tipo de IED especializado em converter sinais analogicos de corrente e tensao dos transformadores de instrumentos em amostras digitais (Sampled Values) para transmissao pela rede. Exemplos: MU_360_1 e MU320E_LAB. |
| **MIB** | Management Information Base (Base de Informacoes de Gerenciamento). Arquivo que define a estrutura e os nomes legiveis dos OIDs de um equipamento SNMP. No NMS, MIBs sao cadastrados na tela de Configuracao e utilizados para traduzir OIDs numericos em nomes compreensiveis (ex: `1.3.6.1.2.1.1.5.0` → `sysName`). |
| **MMS** | Manufacturing Message Specification (Especificacao de Mensagens de Manufatura). Protocolo de comunicacao IEC 61850 baseado em TCP/IP para acesso a dados e servicos dos IEDs em tempo real. Opera sobre o Station Bus e permite leitura/escrita de variaveis, transferencia de arquivos e reports. |
| **NMS** | Network Monitoring System (Sistema de Monitoramento de Rede). A plataforma de software sendo projetada nestes wireframes. Monitora subestacoes IEC 61850 em tempo real, coletando dados de protocolos GOOSE, SV, PTP, MMS e SNMP, gerando alarmes e permitindo analise de topologia e metricas. |
| **OID** | Object Identifier (Identificador de Objeto). Sequencia numerica hierarquica que identifica de forma unica uma variavel monitorada via SNMP (ex: `1.3.6.1.2.1.1.3.0` identifica o `sysUpTime` de um equipamento). A traducao de OIDs numericos para nomes legiveis e feita por MIBs. |
| **PCAP** | Packet Capture (Captura de Pacotes). Arquivo binario contendo uma gravacao do trafego de rede capturado durante um evento ou anomalia. No NMS, PCAPs sao gerados automaticamente quando alarmes sao disparados e podem ser baixados para analise forense. |
| **Process Bus** | Barramento de processo. Rede de comunicacao Layer 2 em uma subestacao IEC 61850 dedicada ao trafego de Sampled Values e GOOSE entre merging units e reles de protecao. Opera com requisitos rigorosos de latencia e determinismo. Exemplos: W4, W01. |
| **PRP** | Parallel Redundant Protocol (Protocolo de Redundancia Paralela). Protocolo de redundancia de rede definido na norma IEC 62439-3 que utiliza duas redes fisicas independentes em paralelo, garantindo comunicacao sem perda de pacotes durante falhas. No NMS, esta pendente de definicao na API. |
| **PTP** | Precision Time Protocol (Protocolo de Tempo de Precisao). Protocolo IEEE 1588 para sincronizacao de relogios em redes Ethernet com precisao de nanossegundos. Essencial em subestacoes IEC 61850 para garantir a correta ordenacao temporal de eventos e Sampled Values. Monitorado na tela de Sincronismo Temporal. |
| **RBAC** | Role-Based Access Control (Controle de Acesso Baseado em Papeis). Modelo de permissoes do NMS onde cada usuario possui um papel (ADMIN, OPERATOR ou VIEWER) que determina quais acoes pode executar no sistema. |
| **Refresh Token** | Token de longa duracao utilizado para obter um novo access token (JWT) quando o atual expira, sem exigir que o usuario faca login novamente. O refresh token e enviado ao endpoint `/api/v1/tokens/refresh`. |
| **Rele de Protecao** | Tipo de IED responsavel por detectar falhas eletricas (curto-circuito, sobrecorrente, etc.) e comandar a abertura de disjuntores para proteger os equipamentos da subestacao. Exemplos: L90_DIG (protecao de linha) e T60_DIG (protecao de transformador). |
| **REST** | Representational State Transfer (Transferencia de Estado Representacional). Estilo arquitetural utilizado pela API do NMS, onde recursos sao acessados via metodos HTTP padrao (GET, POST, PUT, PATCH, DELETE) em endpoints organizados por entidade. |
| **SCD** | Substation Configuration Description (Descricao de Configuracao de Subestacao). Arquivo XML no padrao IEC 61850 que descreve a configuracao completa de uma subestacao: IEDs, redes de comunicacao, fluxos GOOSE/SV, datasets e topologia. E o documento central do NMS — toda a topologia e os pontos de monitoramento sao derivados dele. |
| **SHA-256** | Secure Hash Algorithm 256-bit (Algoritmo de Hash Seguro de 256 bits). Funcao criptografica que gera uma impressao digital unica de um arquivo. No NMS, o hash SHA-256 e usado para detectar uploads duplicados de arquivos SCD/CID. |
| **SNMP** | Simple Network Management Protocol (Protocolo Simples de Gerenciamento de Rede). Protocolo padrao para monitoramento de equipamentos de rede (switches, roteadores). No NMS, os agentes de monitoramento coletam dados SNMP dos switches da subestacao via polling periodico, traduzindo OIDs com auxilio de MIBs. |
| **Station Bus** | Barramento de estacao. Rede de comunicacao IP/MMS em uma subestacao IEC 61850 que conecta IEDs, servidores e sistemas SCADA. Opera sobre TCP/IP e transporta dados MMS, acesso remoto e comunicacao entre sistemas. Exemplos: W1, W2. |
| **Subestacao** | Instalacao eletrica que transforma niveis de tensao e distribui energia. No contexto do NMS, e a instalacao fisica onde estao os IEDs, redes de comunicacao e equipamentos monitorados. Cada SCD descreve uma subestacao especifica. |
| **SV** | Sampled Values (Valores Amostrados). Protocolo IEC 61850 para transmissao de amostras digitais de corrente e tensao em tempo real, tipicamente a 4800 amostras por segundo (para sistemas 60 Hz). Publicados por merging units e consumidos por reles de protecao via Process Bus. A subestacao de exemplo possui 6 streams SV. |
| **TmAcc** | Time Accuracy (Acuracia Temporal). Data Attribute do no logico LTMS que indica a precisao da sincronizacao temporal de um IED, medida em nanossegundos. Valores baixos indicam boa sincronizacao; valores altos indicam degradacao (ex: > 1000 ns = degradado, > 10000 ns = sem sincronismo). |
| **TmSrc** | Time Source (Fonte de Tempo). Data Attribute do no logico LTMS que indica a fonte de sincronizacao temporal de um IED (ex: PTP, GPS, NTP). Se a fonte nao for PTP, pode indicar perda de sincronismo com o Grandmaster Clock. |
| **UTC** | Coordinated Universal Time (Tempo Universal Coordenado). Padrao de tempo global utilizado como referencia pela API do NMS. Todos os timestamps retornados pela API estao em UTC (ex: `2026-02-19T14:30:00Z`). O frontend converte para o fuso horario local do usuario antes de exibir. |
| **UUID** | Universally Unique Identifier (Identificador Unico Universal). Formato padrao de identificador de 128 bits usado pelo NMS para referenciar entidades (alarmes, SCDs, MIBs, usuarios, PCAPs). Exemplo: `a1b2c3d4-e5f6-7890-abcd-ef1234567890`. |
| **VLAN** | Virtual Local Area Network (Rede Local Virtual). Tecnica de segmentacao logica de redes Ethernet. No contexto da subestacao, VLANs separam o trafego de diferentes protocolos e prioridades na mesma infraestrutura fisica (ex: VLAN `0x044F` para streams SV, `0x0837` para GOOSE). |
